# DBC Governance

## Goals
- stable decoding contract
- deterministic decode semantics
- controlled evolution

## Versioning
- breaking change → major bump
- new signals → minor bump
- comment-only → patch bump

## Review requirements
- evidence trace bundle demonstrating the signal
- unit/scaling verification and plausibility checks
- regression check against Golden Trace Suite

## Release artifacts
- dbc file hash
- change log
- mapping to signal registry versions
